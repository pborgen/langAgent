# Project Planning Docs

This folder is the implementation planning source of truth for the Customer Support Agent SaaS.

## Documents

- `PROJECT_GOALS.md`: product vision, users, outcomes, non-goals, success metrics
- `ARCHITECTURE_AND_STANDARDS.md`: current architecture, target architecture, data model, engineering standards
- `IMPLEMENTATION_PLAN.md`: phase-by-phase roadmap with deliverables and definition of done
- `EXECUTION_PLAN.md`: dated sprint plan with concrete tasks, exit criteria, and immediate next actions
- `FEATURE_BACKLOG.md`: prioritized features with user stories and acceptance criteria
- `SPRINT_1_BOARD.md`: sprint-level task board with estimates, dependencies, and status

## How To Use These Docs

1. Start in `PROJECT_GOALS.md` to align on why we are building this product.
2. Use `ARCHITECTURE_AND_STANDARDS.md` before designing any new feature.
3. Use `EXECUTION_PLAN.md` for current sprint work and deadlines.
4. Pull active tasks from `IMPLEMENTATION_PLAN.md` and `FEATURE_BACKLOG.md`.
5. During implementation, update status and decisions in these files (not ad hoc notes).
